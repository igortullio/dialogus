# ADR-002: Scholarly grounded agent posture

## Status

Accepted

## Date

2026-04-24

## Context

The conversational agent can take multiple postures in terms of tone, use of citations, and whether it adopts any persona. The market scan identified two camps: (a) Character.AI-style roleplay (persona, no citations), (b) tutorial-tier generic RAG chat (citations optional, tone neutral-to-bland). A third niche — "serious scholar's companion" with neutral tone and mandatory citations — is underserved.

The owner's use case is academic-style reading (questions about plot, themes, symbols, characters with citation verification), not roleplay or casual chat. Portfolio value also favors the grounded posture — the engineering story is "citations you can verify", not "characters you can flirt with".

## Decision

The V1 agent adopts a **scholarly, grounded posture**:

- Tone: neutral, academic, factual.
- No persona adoption — the agent speaks as itself, never as a character in the book.
- Every non-trivial claim must include at least one citation `{book, chapter, excerpt}` resolved server-side from persisted chunk IDs.
- When retrieval returns nothing usable, the agent refuses rather than hallucinates.
- System prompt explicitly instructs the agent to search before answering, to quote only short excerpts, and to respect the spoiler cap set on the thread.

## Alternatives Considered

### Alternative 1: Reading companion (conversational)

- **Description**: Conversational "book club friend" tone; maintains session state about reading progress; suggests things to watch for; citations optional.
- **Pros**: Warmer UX; discovery-friendly for users who don't yet know what to ask.
- **Cons**: Dilutes the grounding guarantee (warm tone invites hedging, which invites hallucination); weaker portfolio story.
- **Why rejected**: contradicts the "verifiable citations" goal that anchors the product's differentiation.

### Alternative 2: Hybrid with explicit modes

- **Description**: UI toggle between "Modo Estudo" (scholarly) and "Modo Companheiro" (conversational), same backend but different system prompts.
- **Pros**: Lets users choose; more flexible.
- **Cons**: Doubles UX-design and prompt-engineering work; ambiguous behavior in mixed-mode threads; mode signalling in the UI adds surface area.
- **Why rejected**: scope creep; V1 already carries enough novelty with spoiler boundary.

### Alternative 3: Persona roleplay

- **Description**: Agent adopts the voice of a character ("speak to me as Don Quixote"). Citations optional; focus is immersion.
- **Pros**: Emotionally engaging; closest to viral/demo appeal.
- **Cons**: Directly competes with Character.AI Books; hard to differentiate without scale; hallucination risk highest (character-voice invites invention).
- **Why rejected**: wrong product for the owner's use case and for the portfolio positioning.

## Consequences

### Positive

- Clear design target for prompt engineering, retrieval heuristics, and UI affordances.
- Strong portfolio story: "grounded AI with verifiable citations".
- Refusal-on-empty-retrieval becomes a measurable quality signal rather than a bug.

### Negative

- Cold or terse for users who want casual discovery.
- Every answer requires retrieval; short informal questions ("what do you think?") get forced into the retrieval pipeline.

### Risks

- **Risk**: agent becomes pedantic and annoying in daily use.
  **Mitigation**: system-prompt tuning during the RAG-agent feature spec, validated against the owner's real questions.
- **Risk**: refusal-on-empty-retrieval fires too aggressively on borderline-retrievable questions.
  **Mitigation**: threshold tuning; fallback response template that's helpful without hallucinating ("I couldn't find a passage that directly supports this — want me to search more broadly?").

## Implementation Notes

- System prompt is drafted in the RAG-agent feature spec, not this PRD cycle.
- Retrieval tool set is determined by the RAG-agent feature spec (semantic search + chapter/character auxiliaries).
- Citations bind to persisted chunk UUIDs; the UI fetches excerpt text server-side.
